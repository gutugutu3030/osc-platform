# fixTests

このディレクトリは、各 Gradle モジュールのテスト追加・修正方針をモジュール単位で固定化した作業指示書を置く。

## 対象モジュール

- osc-core
- osc-transport-udp
- osc-adapter-cli
- osc-adapter-mcp
- osc-adapter-webui
- osc-cli
- osc-codegen
- osc-gradle-plugin

## 共通ルール

1. テスト以外の実装コードは変更禁止とする。
2. 変更可能なのは `src/test` 配下、必要なテスト補助コード、テスト用リソース、テストに付随する最小限の Markdown のみとする。
3. `src/main` の Kotlin / Gradle 実装を変更しない限りテストが成立しない場合は、その場で無理に直さずレポートを作成する。
4. 追加テストは正常系と異常系を最低 1 件ずつ含める。
5. 既存テストの削除や弱体化は、誤ったテストであると確認できた場合に限る。

## 標準作業手順

1. 対象モジュールの Markdown を読む。
2. `現状` を確認し、既存テストで守れている境界と守れていない境界を把握する。
3. `追加するテスト` を上から順に実装する。
4. `修正する既存テスト` があれば、期待値と責務を揃える。
5. `重複整理` を必要最小限で行う。
6. モジュール単位の `推奨検証コマンド` を実行する。
7. 失敗が `src/main` 修正なしで解消できない場合は `要レポート条件` に従って報告する。

## 失敗時レポートの最低項目

- モジュール名
- 失敗したテスト名
- 実行コマンド
- 失敗ログの要約
- `src/main` を変更しないと解消できないと判断した理由
- 回避策があればその候補
